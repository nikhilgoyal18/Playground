### EP211: How the JVM Works `Engineering`
- javac compiles source code into platform-independent bytecode stored as .class files, JARs, or modules
- Class loading uses parent delegation: Bootstrap handles core JDK classes, Platform covers extensions, System loads application code
- Bytecode linking has three steps: Verify (checks bytecode safety), Prepare (allocates static fields with default values), Resolve (converts symbolic references to direct memory addresses)
- JVM memory is shared (Heap, Method Area) across threads; per-thread resources are JVM stack, PC register, and native method stack
- Execution combines interpretation and JIT compilation: interpreter runs bytecode directly; when a method is called multiple times, JIT compiler converts it to native machine code stored in the code cache
