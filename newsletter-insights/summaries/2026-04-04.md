### EP209: 12 Claude Code Features Every Engineer Should Know `AI/ML`
- **CLAUDE.md** is a project memory file Claude reads at session start — use it to define custom rules, conventions, and context for your project
- **Plan Mode** lets Claude plan before acting; you can review the plan before any code is changed
- **Hooks** run custom shell scripts on lifecycle events like `PreToolUse` or `PostToolUse` — enabling automated guardrails and workflows
- **MCP** connects Claude to external tools (databases, third-party services); **Plugins** extend it further with bundled skills, MCPs, and hooks
- **Subagents** spawn parallel agents for complex tasks — divide large multi-step workflows and run them simultaneously
- **Checkpoints** provide automatic project snapshots to revert to if something goes wrong; **Compaction** compresses long conversations to save tokens

### Agentic RAG vs Traditional RAG `AI/ML`
- Traditional RAG has static retrieval with limited adaptability; Agentic RAG adds AI agents that make decisions, select tools, and refine queries dynamically
- The agent uses both short-term and long-term memory to track query context and formulate a retrieval strategy
- Data fetching can leverage vector search, multiple agents, and MCP servers in parallel to gather relevant information
- The agent combines retrieved data with the query and system prompt, enabling more accurate and context-aware responses than static pipelines
